# Plan: README.ja.md 日本語全面見直し

## Context

v0.4.x の機能追加に伴い README.ja.md を大幅に更新したが、機械翻訳調・不正確な表現・
文体の不統一が多数残存している。ユーザーから「日本語が変」と指摘があったため、
全文を精査して自然な技術文書品質に改善する。

---

## 調査結果（14 箇所の問題を特定）

### High 優先度（実装との乖離あり）
| 行 | 現在 | 問題 | 修正案 |
|----|------|------|--------|
| ~135 | export フラグテーブルに `--upper` あり | 構文シグネチャに `--upper` が含まれない不一致 | シグネチャに `[--upper]` を追加 |

| 行 | 現在 | 修正案 |

| 3 | 「統合する単一バイナリ Go CLI。」 | 「統合した単一バイナリの Go CLI。」 |

- 「有効な間に」 → 「有効なうちに」（自然な慣用句）
- 長すぎる 1 文を 2 文に分割

### 必須修正（行番号は概算。実ファイルを読んで正確な箇所を特定すること）

1. **冒頭説明 (行3)**: `AWS Parameter Store と Secrets Manager を統合する単一バイナリ Go CLI。`
   → `AWS Parameter Store と Secrets Manager を統合して操作する Go CLI。`
   （「単一バイナリ」は本質的な売りではないので削除。シンプルな説明文に）

2. **bundr put 説明文**: `値を保存する。`
   → `値を保存します。`

3. **bundr get 説明文**: `値を取得する。`
   → `値を取得します。`

4. **bundr export 説明文**: `プレフィックス配下のパラメータを環境変数形式で出力する。`
   → `プレフィックス配下のパラメータを環境変数形式で出力します。`

5. **bundr export 構文シグネチャ**: `[--recursive]` の後に `[--upper] [--no-flatten] [--array-mode join|index|json]` を追加

6. **bundr export フラグ `--format` 説明**: `出力フォーマット（必須）` に修正（現在は空欄）

7. **bundr ls 説明文**: `プレフィックス配下のパラメータ一覧を表示する。`
   → `プレフィックス配下のパラメータ一覧を表示します。`

8. **bundr exec 説明文**: `複数の \`--from\` を指定でき、後のプレフィックスが優先される。`
   → `複数の \`--from\` を指定でき、後のプレフィックスが優先されます。`

9. **bundr jsonize 説明文**: `JSON にまとめて stdout に出力する`
   → `JSON オブジェクトに変換して stdout に出力します`

10. **jsonize テーブル `--frompath` 説明**: `読み込み元の SSM プレフィックスまたはリーフ ref（複数指定可）`
    → `読み込み元の SSM プレフィックスまたは末端パラメータ（リーフ ref）（複数指定可）`

11. **jsonize テーブル `--to` 説明**: `保存先の ref（省略時は stdout に出力）`
    → そのまま維持（問題なし）

12. **cache 説明文**: `パラメータ一覧キャッシュを管理する（補完高速化のため自動管理）。`
    → `パラメータ一覧キャッシュを管理します（Tab 補完を高速化するため自動更新されます）。`

13. **completion 説明**: `Tab 補完はパラメータ階層を1段ずつたどれる:`
    → `Tab 補完でパラメータ階層を1段ずつたどれます:`

14. **詳細トピック JSON フラット化の説明文**:
    `エクスポート時に区切り文字（デフォルト \`_\`）でキーをフラット化する。`
    → `エクスポート時に区切り文字（デフォルト \`_\`）でキーを展開して環境変数に変換します。`

15. **既知制限の文**: `クレデンシャルが有効な間に事前キャッシュを作成しておくことを推奨:`
    → `クレデンシャルが有効なうちに、事前キャッシュを作成しておくことを推奨します:`

16. **設定優先順位の説明文**: `後の設定が先の設定を上書きする` を含む文
    → `優先度が高いほど後の番号に記載されています` に修正

17. **`.bundr.toml` セクション説明文** が「配置することで〜可能です。」と長い 1 文の場合:
    → 「配置します。プロジェクト固有の設定が可能です。」に分割

18. **タグスキーマテーブル `cli-store-mode` 説明**: `ストレージモード（\`get\`/\`export\` 時の自動デコードに使用）`
    → `ストレージモード（bundr が \`get\`/\`export\` 時にデコード方式を自動判定）`

### 文体統一（全体）
- `～する。` 形式の行末をすべて `～します。` に変換
- `～できる。` → `～できます。`
- コードブロック内のコメント（`#` で始まる行）は変更しない
- セクション見出し（`##` `###`）は変更しない
- テーブルのヘッダー行は変更しない
